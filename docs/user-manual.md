# ユーザーマニュアル

ランチくじ（Lunch Roulette）の使い方を説明する。システムの概要は
[overview.md](overview.md) を参照。

## 1. 初回セットアップ（お店リストの取り込み）

1. **Google マップで保存済みリストを作る**
   Google マップでお店を検索し、「保存」からリスト（例: 「会社近くのランチ」）
   に追加する。エリアごとにリストを分けておくと後で便利。
2. **Google Takeout で CSV としてエクスポートする**
   Google Takeout の「保存済みの場所」からリストごとに CSV をダウンロードする。
3. **`データ管理` 画面（`/data`）で CSV を取り込む**
   ファイル選択から CSV を選ぶとお店が読み込まれる。CSV のファイル名が
   そのままエリア名になるので、わかりやすい名前（例: `渋谷.csv`）にしておく。
   同じお店は自動的に重複登録されない。

## 2. タグ付けの使い方

`データ管理` 画面（`/data`）で、お店ごとに以下を編集できる。

- **ジャンル** — 「ラーメン」「カフェ」など。手動編集のほか、
  「Places情報を取得」を行うと自動でも付与される。
- **気分** — 「がっつり」「さっぱり」など。こちらは手動で付ける。

行ごとの削除ボタンで個別削除、画面上部の「すべて削除」で全データを消去できる。

### Places情報の取得

「Places情報を取得」ボタン（個別・一括）を押すと、Google Places API から
座標・評価・レビュー件数・営業時間などを取得できる。事前に `設定` 画面で
Google Maps APIキーの登録が必要。

## 3. ランチ提案の使い方

トップ画面（`/`）で以下を行う。

1. **エリア**・**ジャンル**・**気分** のタグをタップして絞り込む。
   「残り営業時間」フィルタで閉店間際のお店を除外することもできる。
2. **並び替え**（ランダム／現在地から近い順／評価順）で一覧表示できる。
   Places情報を取得済みのお店は地図にも表示される。
3. 「ランダムに選ぶ」ボタンを押すと、条件に合うお店の中から1件が
   「今日のおすすめ」として表示される。
4. お店の Google マップリンクから経路確認などができる。

タグを何も選ばない場合は、登録済みの全店舗が対象になる。

## 4. データのバックアップ・復元

`データ管理` 画面（`/data`）で以下ができる。

- **JSON エクスポート** — 全データを JSON としてダウンロードする。
  定期的なバックアップを推奨。
- **JSON インポート** — JSON ファイルを読み込むと、現在のデータを
  **すべて置き換えて**復元する（既存データとのマージではない）。

## 5. 設定画面

`設定` 画面（`/settings`）では以下が行える。

- **Google Maps APIキーの登録** — Places情報取得・地図表示に必要。
- **テーマの切り替え** — ライト／ダーク／端末設定に合わせる。
- **昼休み時間の設定** — 「残り営業時間」フィルタの基準（既定60分）。
- **バージョン確認** — 現在のバージョンとリリース日を表示（自動採番）。
- **Googleログイン／ログアウト（クラウド同期）** — ホワイトリスト登録済みの
  アカウントでログインすると、データがFirestoreと自動で双方向同期される。
  対象外アカウントはログインエラーになる。ログアウト中は通信しない。

## 6. トラブルシューティング

- **CSV を取り込んでもお店が表示されない**
  `Title`（店名）/`Note`（メモ）/`URL` に近い列名になっているか確認する。
  店名の列が無い行はスキップされる。
- **ジャンルが実際と違う内容になっている**
  自動反映は参考情報なので、`データ管理` 画面で手動修正する。
- **Googleログインでエラーになる**
  ホワイトリスト未登録のアカウントではログインできない仕様。
- **データが消えてしまった**
  ブラウザのデータ（localStorage）を消去すると、バックアップの JSON が
  無い限り復元できない。日頃から JSON エクスポートをしておくこと。
