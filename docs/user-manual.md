# ユーザーマニュアル

ランチくじ（Lunch Roulette）の使い方を説明する。システムの概要は
[overview.md](overview.md) を参照。

## 1. 初回セットアップ（お店リストの取り込み）

1. **Google マップで保存済みリストを作る**
   Google マップでお店を検索し、「保存」からお好みのリスト
   （例: 「会社近くのランチ」）に追加していく。エリアごとに
   リストを分けておくと、後で CSV を分けて取り込む際に便利。
2. **Google Takeout でリストを CSV としてエクスポートする**
   Google Takeout から「保存済みの場所」を選択してエクスポートし、
   リストごとの CSV ファイルを取得する。
3. **アプリの `データ管理` 画面（`/data`）で CSV を取り込む**
   画面上のファイル選択から CSV を選ぶと、自動でお店が読み込まれる。
   - **CSV のファイル名がそのままエリア名として登録される**ので、
     わかりやすいファイル名（例: `渋谷.csv`）にしておくと良い。
   - 同じお店（同一エリア内で URL または店名が一致）は重複登録されない。

## 2. タグ付けの使い方

`データ管理` 画面（`/data`）で、取り込んだお店ごとに以下を編集できる。

- **ジャンル** — 「ラーメン」「カフェ」など。手動で自由に編集できるほか、
  後述の「Places情報取得」を行うとGoogle Places APIの公式ジャンルから
  自動でも反映される（手動タグとは統合され、上書きされない）。
- **気分** — 「がっつり」「さっぱり」など、そのときの気分に合わせたタグ。
  自動反映はされないため、手動で付ける。

お店ごとの行にある削除ボタンで個別削除、画面上部の「すべて削除」で
全データを消去できる（誤操作防止のため確認が入る）。

### Places情報の取得

`データ管理` 画面で、お店ごとの「Places情報を取得」ボタン（または一括取得）
を押すと、Google Places API から座標・評価・レビュー件数・営業時間などを
取得してお店の情報に紐付ける。取得にはあらかじめ `設定` 画面で
Google Maps APIキーを登録しておく必要がある。取得は店ごとに1回のみ
行われ、結果はキャッシュされる。一括取得時はAPIのレート制限に配慮し、
一定間隔を空けながら順に処理する。

## 3. ランチ提案の使い方

トップ画面（`/`）で以下を行う。

1. 絞り込みたい **エリア**・**ジャンル**・**気分** のタグをタップして選択する
   （同じ軸内の複数タグは OR 条件、軸をまたぐと AND 条件で絞り込まれる）。
   「残り営業時間」フィルタで、閉店間際のお店を除外することもできる。
2. 一覧は **並び替え**（ランダム／現在地から近い順／評価順）で表示できる。
   Places情報を取得済みのお店は地図上にも表示される。
3. 「ランダムに選ぶ」ボタンを押すと、評価・レビュー件数・距離・直近に
   選ばれたかどうかを加味したスコアリングにより、条件に合うお店の中から
   1件が「今日のおすすめ」として表示される（連続して同じ店になりにくい）。
4. 表示されたお店の Google マップリンクから、そのまま経路確認などができる。

タグを何も選ばない場合は、登録済みの全店舗が対象になる。

## 4. データのバックアップ・復元

`データ管理` 画面（`/data`）で以下ができる。

- **JSON エクスポート** — 全データを日時入りのファイル名で JSON として
  ダウンロードできる。定期的にバックアップを取っておくことを推奨。
- **JSON インポート** — エクスポートした JSON ファイルを読み込むと、
  現在のデータを**すべて置き換えて**復元する（既存データとのマージではない
  点に注意）。

## 5. 設定画面

`設定` 画面（`/settings`）では以下が行える。

- **Google Maps APIキーの登録** — Places情報取得・地図表示に必要。
  入力したキーはこの端末のブラウザ内に保存される。
- **テーマの切り替え** — ライト／ダーク／端末設定に合わせる、の3種類。
- **昼休み時間の設定** — 「残り営業時間」フィルタの基準時間として使用（既定60分）。
- **バージョン確認** — 現在のアプリのバージョンとリリース日を確認できる。
  バージョンはリリース時に自動採番されるため、手動では変更しない。
- **Googleログイン／ログアウト（クラウド同期）** — ホワイトリスト登録済みの
  特定アカウントに限り、Googleログインするとこの端末のデータと
  Firestore上のデータが自動的に双方向同期される。対象外のアカウントで
  ログインした場合はエラーになり、同期は行われない。ログアウト中は
  一切のサーバー通信が発生しない。

## 6. トラブルシューティング

- **CSV を取り込んでもお店が表示されない**
  CSV の列名が想定外の可能性がある。`Title`（店名）/`Note`（メモ）/`URL`
  に近い列名（title/name, note/comment, url/link など）になっているか
  確認する。店名の列が無い行はスキップされる。
- **ジャンルが実際と違う内容になっている**
  Places API取得結果からの自動反映はあくまで参考情報なので、
  `データ管理` 画面で手動修正すれば良い。
- **Googleログインでエラーになる**
  ホワイトリストに登録されていないアカウントではログインできない仕様。
  対象アカウントか確認する。
- **データが消えてしまった**
  ブラウザのデータ（localStorage）を消去した場合、バックアップの JSON
  が無いと復元できない。日頃から JSON エクスポートを行っておくこと。
