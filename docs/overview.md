# システム概要

ランチくじ（Lunch Roulette）は、Google マップに保存したお店リストから「今日のランチ」を
ランダムに提案してくれる、自分専用のシンプルなツール。

技術的な詳細は [ARCHITECTURE.md](../ARCHITECTURE.md) を、
使い方は [user-manual.md](user-manual.md) を参照。

## 目的・対象ユーザー

普段から Google マップに気になるお店を保存しているが、いざランチの時間に
なると「結局どこにするか」迷ってしまう、という悩みを解決するための
個人利用ツール。ジャンルや気分でお店を絞り込み、最後は「ランダムに選ぶ」
ボタンに決めてもらう。

## 主要機能

- **CSV 取り込み** — Google Takeout でエクスポートした保存リスト（CSV）を
  そのまま読み込める。ファイル名がエリア名として扱われる。
- **店舗情報の自動取得・ジャンル自動反映** — Google Places API で座標・評価・
  営業時間などの店舗情報を取得すると、Places の公式ジャンル情報から
  「ラーメン」「カフェ」などのジャンルタグも自動反映される（手動タグとの和集合）。
- **タグ付け・編集** — ジャンル・気分タグは取り込み後に自由に編集できる。
- **絞り込み & ランダム提案** — エリア・ジャンル・気分・残り営業時間で絞り込み、
  地図表示や距離順／評価順ソートを見ながら、該当するお店の中から
  1件をランダム（またはスコアリングによるおすすめ）で提示する。
- **バックアップ / 復元** — 全データを JSON ファイルとして書き出し・
  読み込みできる。
- **クラウド同期（任意）** — ホワイトリスト登録済みの特定アカウントに限り、
  Googleログインすると Firestore へデータを双方向同期できる。

## 重要な制約

- **既定ではデータはこの端末のブラウザ内（localStorage）にのみ保存される。**
  サーバーには自動送信されない。ブラウザのデータを消去すると
  リストも失われるため、定期的な JSON バックアップを推奨する。
- Googleログインしない限り、別の端末・ブラウザとはデータが同期されない。
  ログインした場合のみ、ホワイトリスト対象アカウントに限定して
  Firestore経由で同期される（オプトイン）。
- 生成AIは一切使用していない。ジャンル自動反映はGoogle Places APIの
  構造化データ（公式ジャンル`types`）に基づくルールベース変換であり、
  ユーザー操作（ボタン押下）を起点にのみ通信する。バックグラウンドでの
  自動通信は行わない。
